Usage: dialtone <command> [options]
       ./dialtone.sh help
       ./dialtone.sh --help
       .\dialtone.ps1 help
       .\dialtone.ps1 --help

Commands:
  install [path]      Install Go toolchain to DIALTONE_ENV (go plugin default)
  ps <option>         List Dialtone processes (all, tracked, tree)
  proc <subcmd>       Process management (ps, stop, logs)
  kill <pid|all>      Kill one Dialtone process tree or all Dialtone wrapper processes
  help                Show this help message

Other commands:
  Any other command is forwarded to src/cmd/dev/main.go via managed Go in DIALTONE_ENV.

Global Options:
  --env <path>        Set DIALTONE_ENV env file path
  --grace <sec>       Seconds to wait before SIGKILL in proc/kill (default: 5)
  --timeout <sec>     Optional. If set, kills the forwarded subprocess tree after N seconds

Process Notes:
  - dialtone wrappers do not auto-kill child processes on shell exit.
  - Nested dialtone subcommands are tracked while running.
  - Use explicit process controls: ps / proc / kill.

Examples:
  ./dialtone.sh install
  ./dialtone.sh ps
  ./dialtone.sh proc ps
  ./dialtone.sh kill all
